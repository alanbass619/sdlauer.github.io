Proof checking:
a modification of Parson's coding problem

HISTORY: zyBooks has an Authoring Tool for Parson's Problems. Lines of code are given in random order and students drag them into the correct order and run to check.

GOAL: Adapt this Authoring Tool (or use it to create another) that can be used for common problems in Math, particulary proofs.

PROS:

  1. Current approach to proofs in PAs is easy to cheat. Parson's problems are checked as a whole.
  2. Proof assessments have had to be carefully manicured to aoivd dependencies.
  3. Can be PAs or CAs.
  4. Research papers suggest this is a good approach.
  5. Cutting edge.

MATH TOOL NEEDS:

  1. Everything in the tool is recognized as code, when building and running.
  2. Checking: Check gives spinning wheel since it's trying to run code.
  3. 1-column and 2-column options
  4. Support \(\LaTeX\): In tool, Latex can be enetered but does not render at Run Player. Prompt does not current support Latex at all.
  5. Prompt: Prompt in Tool should support images to be effective for proofs.
  6. Dependencies: Math proof steps often do not have a strict order.
  7. Hints for errors
  8. Distract or not distract?
  9. Distractor drawer
