Proof blocks: A modification of Parson's coding problem

HISTORY: zyBooks has an Authoring Tool for Parson's Problems. Lines of code are given in random order and students drag them into the correct order and run to check

GOAL: Adapt parsons builder tool (or use it to create another) that can be used for common problems in math, particulary proofs

MVP (proofs):

  1. One-column
  2. Latex throughout
  3. Static prompt: "Below are the steps of the theorem proof. Drag the steps into the correct order."
  4. One feedback per statement, each step has two boxes to fill in - Statement/Feedback
  5. Dependencies ("Dependency" checkbox in builder, like "Editable" in Parson's tool)
  6. Front end - Statement space as sandbox
  7. PA - immediate feedback with feedback/redbox for incorrect drag
  8. CA - on check give "The proof not complete, all steps should be used." or feedback from first incorrect step
  9. CA - Prefer not to have randomization, scrambbling Levels would be nice

MVP (math and engineering):

  1. One-column/Two-column option with headers for each column
  2. Feedback - "The statement and justification do not match.", plus one feedback per line
  3. Dynamic prompt
  4. Images in prompt

NOT NEEDED FROM PARSON'S TOOL:

  1. Language
  2. Files
  3. Editable
  4. Indents
  5. Tests
  6. Add distractor

PROS TO BLOCK PROOFS:

  1. Current proofs PAs are easy then CAs are hard, block proofs are checked as a whole, too many iterations to guess, puzzle
  2. Supports dependencies, current proof assessments carefully made to avoid dependencies
  3. Can be PAs or CAs, keep PAs in place, add as CAs?
  4. Research papers, level of difficulty (little error feedback), student feedback
  5. Cuttinging edge, started with cards, some schools use it (by altering code?) but could only find Cengage for autograded proofs (not block): Cengage blog with proof demo, no Pearson, no McGraw-Hill
  6. Prototype is dynamic, many different formats
